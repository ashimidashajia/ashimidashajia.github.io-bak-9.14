提取android5.0升级包中得system.img镜像内容

提取android5.0升级包中得system.img镜像内容

标签： 刷机

android5.0的zip包解压之后，包含：

install
boot.img
file_contexts
META-INF
system.new.dat
system.patch.dat
system.transfer.list

system的内容包含在了system.new.dat，所以我们需要对这个文件进行解包操作。system.new.dat文件实际上是由system.transfer.list描述的一个稀疏数组。

2
117569
0
0
erase 2,0,288358
new 32,0,32767,32768,32770,32841,32843,33344,65535,65536,65538,98304,98306,98377,98379,98880,125757,131072,131074,163840,163842,163913,163915,196608,196610,229376,229378,229449,229451,262144,262146,262647,288357

第一行的”2”代表的是该文件的版本号，网络上很多文章中提到得sdat2img，其实是针对版本”1”的，所以不能使用，这里提供一个适合”2”版本的python脚本。复制以下代码，保存为sdat2img.py.放入到解压的升级包中，然后执行：

chmod +x sdat2img.py 
sdat2img.py system.transfer.list system.new.dat new_system.img
sudo mount -t ext4 -o loop,ro,noexec,noload new_system.img /mnt/rom

在/mnt/rom目录下即可看到system目录的内容.
